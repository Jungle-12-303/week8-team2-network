## Review

## 문서 작성 원칙

- 새 문서를 만들면 상위 인덱스 링크도 함께 갱신한다.
- 계획 문서는 `plan/`, 학습 문서는 `study/`, 리뷰 문서는 `Review/`에 둔다.
- 구현 기록은 문제를 먼저 쓰고, 학습과 해결 과정을 이어서 적는다.
