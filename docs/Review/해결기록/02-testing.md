# 테스트 개선 기록

## 문제

프로젝트에서 어떤 테스트가 부족했는지 정리한다.

예시:

- unit test와 integration test가 구분되지 않는 문제
- smoke test 의존도가 높은 문제
- 실패 원인을 바로 추적하기 어려운 문제

## 학습

이 문서에서는 아래 개념을 정리한다.

- 테스트 피라미드
- 기능 테스트와 회귀 테스트의 차이
- 자동화된 검증이 왜 중요한지

## 해결

여기에 실제 적용한 방식을 적는다.

- 어떤 테스트 파일을 추가했는지
- 어떤 시나리오를 우선순위로 잡았는지
- 실패 로그를 어떻게 읽기 쉽게 만들었는지

## 검증

여기에 확인한 내용을 적는다.

- 빌드가 재현 가능한지
- smoke test가 실제로 통과하는지
- 엣지 케이스가 커버되는지

## 남은 과제

- unit test 확대
- 실패 케이스 자동 수집
- CI 연동
