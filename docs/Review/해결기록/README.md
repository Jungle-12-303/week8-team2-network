# 해결기록 사용 가이드

이 폴더는 아직 해결되지 않은 문제를 포함해, 앞으로 프로젝트에서 발견한 부족한 점을
정리하고 학습 및 개선 과정을 남기기 위한 공간이다.

지금 단계에서는 세부 해결 사례를 미리 채워 넣지 않고, 이후 기록을 쌓기 위한
기본 골격만 둔다.

## 이 폴더를 쓰는 기준

- 프로젝트에서 부족한 점을 발견했을 때
- 관련 개념을 학습하고 싶을 때
- 구현 전에 정리해야 할 판단 기준이 생겼을 때
- 해결 과정과 결과를 나중에 다시 참고하고 싶을 때

## 작성 원칙

- 한 문서는 한 주제만 다룬다.
- 아직 해결되지 않은 내용도 기록할 수 있다.
- 해결 전에는 `문제`, `왜 중요한가`, `학습 메모`, `진행 중인 방향` 정도만 적어도 된다.
- 해결 후에는 `해결 방법`, `검증 결과`, `남은 과제`를 추가한다.
- 코드 변경이 있으면 관련 파일과 커밋을 함께 적는다.

## 추천 문서 흐름

1. 문제를 짧게 정의한다.
2. 어떤 부분이 부족한지 적는다.
3. 관련 개념을 학습한다.
4. 적용 가능한 해결 방향을 메모한다.
5. 실제 해결을 마친 뒤 결과를 보강한다.
6. 남은 한계나 다음 개선점을 적는다.
