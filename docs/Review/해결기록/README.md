# 해결기록

이 폴더는 우리 프로젝트를 점검하면서 발견한 부족한 점을 정리하고, 학습과 개선 과정을 남기는 곳이다.

## 목적

- 부족한 점을 감정이 아니라 기준으로 설명한다.
- 무엇을 학습했는지 남긴다.
- 어떻게 해결했는지 구현 관점으로 기록한다.
- 다시 같은 문제가 생겼을 때 빠르게 참고할 수 있게 한다.

## 문서 작성 규칙

- 한 문서에는 하나의 주제만 담는다.
- 각 문서는 `문제 -> 학습 -> 해결 -> 검증 -> 남은 과제` 순서로 쓴다.
- 코드 수정이 있으면 해당 파일과 커밋 기준을 같이 적는다.
- 테스트 결과가 있으면 가능하면 실행 명령과 관찰 결과를 함께 남긴다.
