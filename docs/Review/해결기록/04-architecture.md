# 아키텍처 개선 기록

## 문제

프로젝트 전체 구조가 이해하기 쉬운지 정리한다.

예시:

- server / db / concurrency 책임이 분리됐는지
- 파일이 너무 커져서 읽기 어려운지
- 새로운 기능을 추가할 때 어디를 고쳐야 하는지

## 학습

이 문서에서는 아래 개념을 정리한다.

- 계층형 구조
- 관심사 분리
- adapter 패턴의 장점

## 해결

여기에 실제 적용한 방식을 적는다.

- 폴더를 어떻게 나눴는지
- 각 모듈이 맡는 책임을 어떻게 줄였는지
- 의존성이 어느 방향으로 흐르는지

## 검증

여기에 확인한 내용을 적는다.

- 코드 경로가 직관적인지
- 새 기능 추가 시 수정 범위가 예측 가능한지
- 문서와 코드 구조가 같은 방향인지

## 남은 과제

- 아키텍처 다이어그램 추가
- 모듈 간 인터페이스 더 엄격하게 정리
- 리팩터링 우선순위 정의
