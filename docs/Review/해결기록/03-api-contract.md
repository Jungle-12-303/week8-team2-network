# API 계약 개선 기록

## 문제

API 응답 형식과 상태 코드가 충분히 명확한지 정리한다.

예시:

- 성공 응답과 실패 응답의 필드가 일관적인지
- HTTP status code가 의미에 맞는지
- 클라이언트가 에러를 해석하기 쉬운지

## 학습

이 문서에서는 아래 개념을 정리한다.

- API contract의 역할
- status code 선택 기준
- 에러 응답 표준화

## 해결

여기에 실제 적용한 방식을 적는다.

- 어떤 endpoint를 유지하거나 바꿨는지
- 어떤 응답 필드를 표준화했는지
- 어떤 에러 메시지 규칙을 정했는지

## 검증

여기에 확인한 내용을 적는다.

- curl 또는 HTTP client로 확인한 응답
- 성공/실패/엣지 케이스 결과
- 문서와 구현이 일치하는지

## 남은 과제

- 버전별 API 분리
- 응답 스키마 문서화
- 에러 코드 매핑 표 보강
