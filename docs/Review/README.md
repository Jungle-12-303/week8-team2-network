# Review Docs

이 폴더는 브랜치 비교, 문제 분석, 해결 기록을 모아두는 공간이다.

## 문서 구조

- [브랜치 비교 리포트](hye-yeon-vs-exp1-comparison.md)
- [해결기록](해결기록/README.md)

## 이 폴더를 쓰는 기준

- 프로젝트에서 부족했던 점을 발견했을 때
- 관련 개념을 새로 학습했을 때
- 어떻게 해결했는지 구현과 함께 남기고 싶을 때
- 나중에 다시 봐도 재현 가능한 형태로 정리하고 싶을 때

## 권장 작성 흐름

1. 문제를 먼저 한 문장으로 적는다.
2. 왜 문제가 되는지 기준을 적는다.
3. 학습한 개념과 참고한 자료를 적는다.
4. 실제로 어떤 방식으로 해결했는지 적는다.
5. 테스트나 검증 결과를 적는다.
6. 남은 한계와 다음 개선점을 적는다.
