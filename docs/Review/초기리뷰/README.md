# 초기 리뷰 반영 기록

이 폴더는 프로젝트 초기에 받은 리뷰와 그 반영 결과를 모아두는 곳이다.

## 목적

- 초기에 어떤 지적을 받았는지 남긴다.
- 그 지적을 어떻게 이해했고 반영했는지 정리한다.
- 같은 리뷰를 다시 받지 않도록 기준을 고정한다.

## 문서 구성

- `REVIEW-비판적-평가.md`: 리뷰에서 받은 비판적 관점 정리
- `REVIEW-비판적-평가2.md`: 추가 평가 또는 후속 리뷰 정리
- `REVIEW-반영결과.md`: 첫 번째 반영 결과
- `REVIEW-반영결과2.md`: 두 번째 반영 결과

## 읽는 순서

1. 비판적 평가 문서를 먼저 읽는다.
2. 어떤 문제가 지적됐는지 확인한다.
3. 반영 결과 문서를 읽어서 해결 방향을 본다.
4. 현재 코드나 문서와 비교해 남은 과제를 확인한다.
