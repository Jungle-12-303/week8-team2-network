# 라우팅과 응답 계약 정리함

## 목적

요청 경로에 따라 어떤 응답을 돌려줄지 계약을 고정한다.

## 범위

- `POST /query` 계약 정리함
- unknown path 처리 정리함
- method not allowed 기준 정리함
- JSON 응답 포맷 정리함
- 헤더 계약 정리함

## 완료 기준

- 성공 응답과 실패 응답의 형식이 고정됨
- 클라이언트가 기대할 수 있는 형태가 명시됨
- 에러 코드 기준이 문서화됨

## 테스트/검증

- `200`, `400`, `404`, `405` 응답을 확인함
- 응답 body 구조를 확인함
