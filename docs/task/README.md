# Task Folder Guide

## 목적

이 폴더는 실제 구현 작업을 카드 형태로 관리하기 위한 공간이다.
계획 문서보다 훨씬 더 세분화된 작업을 담는다.

## 사용 원칙

- 하나의 파일 = 하나의 작업 카드
- 카드 하나는 가능한 한 작게 유지
- 제목은 `~함` 형태를 권장
- 완료 조건과 검증 방법을 반드시 적기
- 구현, 테스트, 문서 수정을 한 카드에 섞지 말 것

## 권장 카드 형식

```md
# 작업 제목

## 목적

## 범위

## 완료 기준

## 테스트/검증

## 의존성

## 메모
```

## 분리 기준

아래 중 하나라도 해당하면 카드를 더 쪼개는 편이 좋다.

- 작업 시간이 길어질 것 같음
- 테스트가 여러 개 필요함
- 다른 사람이 병렬로 할 수 있음
- 구현과 리팩터링이 섞여 있음
- 문서 수정까지 포함되면서 범위가 커짐

## 추천 흐름

1. `plan`에서 큰 방향을 정함
2. `task`로 구현 단위를 분해함
3. 카드별로 완료 기준을 적음
4. 작업이 끝날 때마다 카드 상태를 갱신함
5. 테스트 결과가 바뀌면 카드와 `07-test-and-quality-plan.md`를 함께 수정함
