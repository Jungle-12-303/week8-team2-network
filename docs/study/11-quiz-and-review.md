# 11. 자기점검 질문

## 핵심 개념
1. 소켓이 file descriptor라는 말은 실제 코드에서 어떤 의미를 가지는가?
2. hostname, IP address, port는 각각 어떤 역할을 하는가?
3. TCP 위에서 HTTP가 동작한다는 말은 요청 처리 관점에서 무슨 뜻인가?
4. DNS는 연결 흐름 어디쯤에서 필요한가?

## 소켓 흐름
1. 서버 쪽 흐름에서 `bind`, `listen`, `accept`는 각각 무엇을 담당하는가?
2. 클라이언트는 왜 `connect`를 호출해야 하는가?
3. `listenfd`와 `connfd`를 혼동하면 어떤 오해가 생길까?

## echo와 tiny
1. echo server와 tiny의 가장 큰 차이는 무엇인가?
2. tiny는 왜 요청 라인을 파싱해야 하는가?
3. 정적 콘텐츠와 동적 콘텐츠는 무엇이 다른가?
4. CGI 프로그램의 출력이 어떻게 브라우저 응답이 될 수 있는가?

## tiny와 proxy
1. tiny는 직접 응답을 만들고, proxy는 주로 무엇을 하는가?
2. proxy는 왜 서버이면서 동시에 클라이언트라고 말하는가?
3. 프록시는 왜 브라우저 요청을 그대로만 복사하면 끝나지 않을 수 있는가?

## 동시성
1. iterative server의 장점과 한계는 무엇인가?
2. 응답하지 않는 서버가 하나 있을 때 순차 프록시는 왜 약한가?
3. 스레드를 쓰면 무엇이 쉬워지고 무엇이 새로 어려워지는가?

## 캐시
1. cache hit과 cache miss는 무엇인가?
2. tiny를 종료한 뒤에도 프록시가 응답할 수 있다면 무엇이 동작한 것인가?
3. 여러 요청이 같은 캐시에 접근할 때 어떤 문제가 생길 수 있을까?

## 복습 질문
1. 소켓 -> echo -> tiny -> proxy가 왜 자연스러운 학습 순서인가?
2. 이번 과제에서 네트워크 물리 계층을 깊게 보지 않아도 되는 이유는 무엇인가?
3. tiny와 proxy를 구현하기 전에 반드시 설명할 수 있어야 하는 개념 3가지를 골라 보자.
