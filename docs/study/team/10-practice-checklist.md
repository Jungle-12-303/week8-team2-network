# 10. 실습 체크리스트

## 1단계: 소켓 이해하기
해봐야 할 것:
- 서버와 클라이언트의 호출 순서를 종이에 적어 본다
- `listenfd`, `connfd`, `clientfd` 차이를 말해 본다

관찰해야 할 것:
- 서버는 포트에서 기다리고, 클라이언트는 먼저 연결을 시도한다
- 연결 후에는 양쪽 모두 읽고 쓸 수 있다

설명할 수 있어야 하는 것:
- 소켓이 file descriptor인 이유
- TCP 연결과 포트의 의미

## 2단계: 에코 서버 이해하기
해봐야 할 것:
- 클라이언트가 보낸 문자열이 그대로 돌아오는 구조를 그려 본다
- echo와 tiny의 차이를 말로 설명해 본다

관찰해야 할 것:
- 서버는 요청 의미를 몰라도 바이트만 주고받을 수 있다
- 요청 해석이 없으면 응답도 단순하다

설명할 수 있어야 하는 것:
- 왜 echo server가 tiny 이전 학습으로 적절한가

## 3단계: tiny 웹서버 이해하기
해봐야 할 것:
- 브라우저에서 정적 파일 요청과 CGI 요청의 흐름을 각각 설명해 본다
- request line, header, response line의 형태를 써 본다

관찰해야 할 것:
- tiny는 요청을 해석해야만 올바른 응답을 만들 수 있다
- 정적 콘텐츠와 동적 콘텐츠는 처리 경로가 다르다

설명할 수 있어야 하는 것:
- `doit`가 맡는 책임
- `parse_uri`가 왜 중요한가
- `adder.c`가 왜 별도 프로그램인지

## 4단계: CSAPP 11장 연습문제 스스로 공부하기
해봐야 할 것:
- 11.6c, 11.7, 11.9, 11.10, 11.11을 읽고 각 문제가 점검하는 개념을 먼저 적는다
- [07-self-study-framework.md](07-self-study-framework.md)의 질문에 답해 본다

관찰해야 할 것:
- 문제를 읽을 때 "정답 찾기"보다 "개념 확인"이 먼저다
- tiny 또는 proxy 구현과 연결되는 포인트가 있어야 한다

설명할 수 있어야 하는 것:
- 각 문제를 왜 지금 푸는지

## 5단계: Proxy Lab 이해하기
해봐야 할 것:
- 브라우저, 프록시, 원서버 사이 흐름도를 직접 그려 본다
- `Basic`, `Concurrency`, `Cache`가 각각 무엇을 검증하는지 말로 설명해 본다

관찰해야 할 것:
- proxy는 양쪽 연결을 모두 다뤄야 한다
- 동시성과 캐시는 tiny에 없던 새 책임이다

설명할 수 있어야 하는 것:
- proxy가 서버이면서 클라이언트인 이유
- 캐시가 왜 단순 전달 이상의 기능인지
