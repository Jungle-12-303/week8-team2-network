# Architect

## 역할

시스템 구조와 기술 설계를 담당하는 역할이다.

## 책임

- API 엔드포인트와 요청/응답 형식을 설계한다.
- 데이터 흐름과 처리 구조를 정의한다.
- 스레드 풀, 작업 큐, 락 전략 같은 핵심 구조를 결정한다.
- 모듈 경계와 책임 분리를 정리한다.
- 구현 시 지켜야 할 기술적 제약을 문서화한다.

## 주요 산출물

- API 설계 문서
- 시스템 구조도 또는 텍스트 구조 정의
- 데이터 흐름 설명
- 기술 선택 근거

## 설계 기준

- 설명 가능성을 해치지 않는 범위에서 단순한 구조를 우선한다.
- 기존 TEAM7 엔진 재사용 전제를 유지한다.
- 서버 계층과 엔진 계층의 결합을 최소화한다.
- 동시성 제어는 우선 안전하게 설계한다.

## 완료 기준

- Developer가 설계를 그대로 구현 단위로 나눌 수 있어야 한다.
- QA가 어떤 동작을 검증해야 하는지 구조상 추적 가능해야 한다.
