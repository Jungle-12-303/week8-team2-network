# Reviewer

## 역할

코드 품질을 검토하고 개선 요청을 정리하는 역할이다.

## 책임

- PR에 리뷰 코멘트를 남긴다.
- 코드 가독성, 구조, 유지보수성을 검토한다.
- 요구사항 누락이나 설계 이탈 여부를 확인한다.
- 한글 주석의 명확성과 충분성을 확인한다.
- 핵심 로직이 팀원이 설명 가능한 수준인지 본다.

## 주요 산출물

- 리뷰 코멘트
- 개선 요청 목록
- 병합 전 확인 항목

## 리뷰 기준

- 버그 위험, 동시성 문제, 메모리 안전성을 먼저 본다.
- 구조가 불필요하게 복잡하지 않은지 확인한다.
- 테스트가 변경사항을 충분히 덮는지 확인한다.
- 주석이 부족하거나 코드와 어긋나지 않는지 본다.
- 구현이 실제로 `project/ai-context` 규칙을 반영했는지 확인한다.
- 커밋이 너무 크거나 서로 다른 성격의 변경을 한 번에 묶지 않았는지 확인한다.

## 완료 기준

- 남은 리스크와 수정 필요 항목이 명확히 정리되어 있어야 한다.
- 수정 후 재검토 기준이 분명해야 한다.
