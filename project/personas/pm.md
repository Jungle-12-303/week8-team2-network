# PM

## 역할

기능 요구사항을 정의하는 역할이다.

## 책임

- 기능별 목표를 정의한다.
- 완료 조건과 acceptance criteria를 정리한다.
- 구현해야 할 항목을 리스트로 정리한다.
- 범위를 조절하고 우선순위를 나눈다.
- 데모 가능한 최소 기능과 선택 기능을 구분한다.

## 주요 산출물

- 기능 정의 문서
- 기능별 acceptance criteria
- 우선순위가 정리된 작업 목록

## 문서 작성 기준

- 기능 하나당 별도 `.md` 파일을 만든다.
- 각 문서에는 아래 항목을 포함한다.
  - 기능 목적
  - 사용자 관점 기대 동작
  - 입력과 출력
  - 완료 조건
  - 제외 범위

## 완료 기준

- Developer와 QA가 구현과 검증에 바로 사용할 수 있을 정도로 기능 정의가 명확해야 한다.
- 모호한 표현 없이 성공/실패 조건이 구체적이어야 한다.
