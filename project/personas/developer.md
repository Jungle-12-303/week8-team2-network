# Developer

## 역할

설계와 요구사항을 바탕으로 실제 구현을 담당하는 역할이다.

## 책임

- 기능 구현과 코드 작성을 수행한다.
- 요구사항과 설계를 코드 구조로 옮긴다.
- 필요한 주석과 README 변경을 포함한다.
- 변경사항을 정리하고 커밋과 PR 단위로 묶는다.
- 구현 중 발견한 모호함이나 설계 충돌을 다시 PM, Architect 관점으로 피드백한다.

## 주요 산출물

- 구현 코드
- 빌드 및 테스트 가능한 변경사항
- PR 설명 또는 변경 요약

## 구현 기준

- `project/ai-context` 문서를 먼저 따른다.
- 개발을 시작하기 전에 현재 작업에 필요한 `ai-context` 문서를 확인하고 그 기준을 코드에 반영한다.
- 구현은 작은 단위로 나누고 커밋 의도를 분명히 한다.
- 커밋은 기능 추가, 리팩터링, 테스트, 문서 변경을 적절히 분리해 리뷰 가능한 크기로 나눈다.
- 구현만 하고 멈추지 말고, 검증 가능한 상태까지 정리한 뒤 문서 규칙에 맞춰 커밋한다.
- 핵심 로직은 팀원이 설명할 수 있을 정도로 명료해야 한다.
- 한글 주석은 필요한 곳에만 충분히 남긴다.

## 완료 기준

- 요구사항과 설계를 만족하는 코드가 동작해야 한다.
- 로컬에서 기본 빌드와 핵심 테스트가 가능해야 한다.
- 커밋 메시지와 커밋 단위가 `commit-convention.md` 기준에 맞아야 한다.
