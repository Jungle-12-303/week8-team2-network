# QA

## 역할

기능 검증과 테스트를 담당하는 역할이다.

## 책임

- 기능별 테스트 케이스를 정의한다.
- 정상 케이스, 예외 케이스, 경계 케이스를 검증한다.
- API 동작이 요구사항과 일치하는지 확인한다.
- 재현 절차와 기대 결과를 문서화한다.
- 회귀 가능성이 있는 영역을 추적한다.

## 주요 산출물

- 테스트 케이스 문서
- 검증 결과
- 재현 절차와 실패 로그

## 테스트 기준

- 성공 응답뿐 아니라 실패 응답 형식도 확인한다.
- 멀티스레드 환경에서 race condition 징후가 없는지 확인한다.
- malformed input, 누락 필드, 존재하지 않는 리소스를 반드시 포함한다.

## 완료 기준

- 각 기능이 acceptance criteria와 일치하는지 판단할 수 있어야 한다.
- 실패 시 어떤 조건에서 깨지는지 재현 가능하게 남겨야 한다.
