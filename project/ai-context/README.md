# AI Context Docs

AI가 프롬프트를 바탕으로 작업하기 전에 먼저 읽어야 하는 문서를 모아둔 폴더다.
코드 생성, 리팩터링, 리뷰, 커밋 메시지 작성 전에 이 디렉터리의 문서를 우선 참조한다.

## 읽는 순서

1. 현재 작업과 직접 관련된 언어/도메인 규칙 문서
2. Git 협업 및 커밋 규칙 문서
3. 과제별 추가 요구사항 문서가 생기면 그 문서

## 현재 문서

- `c-style.md`: C 코딩 스타일, 메모리 관리, 헤더 규칙, 함수 작성 규칙
- `commit-convention.md`: Git 브랜치 전략, 커밋 메시지, PR 작성 규칙

## AI 작업 원칙

- 개발을 시작하기 전에 현재 작업과 관련된 `ai-context` 문서를 반드시 먼저 읽는다.
- 구현 중 의사결정은 `ai-context` 규칙을 기본값으로 삼고, 예외가 있으면 이유를 남긴다.
- 코드를 작성한 뒤에는 커밋까지 진행하는 것을 기본 흐름으로 본다.
- 커밋은 한 번에 크게 묶지 말고, 의미와 검토 범위가 분명한 단위로 나눈다.
- 커밋 메시지는 `commit-convention.md` 규칙을 따른다.
- 문서 간 규칙이 충돌하면 더 구체적인 문서를 우선한다.
- 명시되지 않은 사항은 기존 코드베이스의 일관성을 우선한다.
- 새로운 규칙을 추가했다면 이 폴더에 문서로 남기고, 변경 이유를 함께 적는다.
- 리뷰 요청 시에는 스타일보다 버그, 회귀, 메모리 안정성, 테스트 누락을 먼저 본다.

## 추가 권장 문서

아래 문서는 프로젝트가 진행되면 이 폴더에 추가하는 것을 권장한다.

- `architecture.md`: 시스템 구조, 모듈 책임, 데이터 흐름
- `build-and-run.md`: 빌드, 실행, 테스트 방법과 필수 환경
- `debugging.md`: 자주 쓰는 gdb, valgrind, sanitizers 사용법
- `assignment-rules.md`: 과제 채점 기준, 금지사항, 제출 형식
