# Health Check

## 목적

`GET /health`는 외부 클라이언트와 운영자가 서버 프로세스가 요청을 받을 수 있는 상태인지 빠르게 확인하기 위한 기능이다.

이 엔드포인트는 DB 조회나 데이터 변경이 아닌, 서버 생존 여부 확인에만 집중한다.

## 사용자 관점 기대 동작

- 클라이언트가 `GET /health`를 호출하면 서버는 짧고 일관된 JSON 응답을 반환한다.
- 서버 프로세스가 정상적으로 요청을 읽고 응답을 쓸 수 있으면 `200 OK`를 반환한다.
- 이 요청은 다른 비즈니스 데이터에 영향을 주지 않는다.

## 요청 입력

### HTTP Request

- Method: `GET`
- Path: `/health`
- Request Body: 없음

### 입력 검증

- request body는 사용하지 않는다.
- 추가 query parameter나 custom header는 최소 구현 범위에서 해석하지 않는다.

## 응답 출력

### 성공 응답

- Status: `200 OK`
- Body:

```json
{
  "ok": true
}
```

### 실패 응답

이 기능은 최소 구현에서 별도 비즈니스 실패 케이스를 두지 않는다. 서버가 응답을 만들 수 없는 수준의 내부 오류는 공통 서버 에러 처리에 따른다.

## Acceptance Criteria

- `GET /health` 요청에 대해 서버가 `200 OK`를 반환한다.
- 응답 본문은 JSON 형식이며 `ok: true`를 포함한다.
- 응답은 DB 엔진 상태와 무관하게 서버 프로세스 생존 여부 확인 용도로 동작한다.
- 이 요청은 데이터 생성, 수정, 삭제를 발생시키지 않는다.
- 동일 요청을 반복 호출해도 결과가 안정적으로 유지된다.

## 구현해야 할 항목

- `/health` 경로를 라우팅한다.
- `GET` 메서드를 인식한다.
- 고정된 JSON 응답을 작성한다.
- 공통 HTTP 응답 헤더와 상태 코드를 일관되게 반환한다.

## 제외 범위

- DB 상세 상태 점검
- thread pool 길이, 큐 길이, 메모리 사용량 같은 운영 통계 응답
- readiness/liveness 분리
- 인증 또는 권한 확인
